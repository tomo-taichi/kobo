# JPY 請求額の端数切り捨てルール

書類（OC・Advance・Final）に印字する **JPY 金額は整数円**で算出する。各商品の単価を `round(unitEur × rate)` で円換算し、行合計・小計・合計を積み上げる（行が小計に一致する）。消費税は `floor(下代小計 × 税率)` を円単位で算出し、合計 = 下代小計 + 消費税。これにより商品ごとに ¥/€ を併記しても行と小計・税・合計が整合し、適格請求書（インボイス制度）の要件（税率ごとの消費税額が基準×税率と一致）を満たす。EUR は1ユーロ単位。

更新履歴:
- 当初 JPY 10円単位 → 2026-06 に 1,000円単位（見栄え優先）→ さらに 2026-06 に**整数円（行単位積み上げ）に変更**。商品ごとの ¥/€ 併記で行と小計が一致しないと請求書として不整合になるため、1,000円丸めは廃止。`docs/adr/0006-document-language-by-currency.md` も参照。
- 注: 社内の Financials ワークシート（画面）と `orders.deposit_amount_jpy`（保存値）は旧 1,000円丸めのロジックが残存。書類・レジャー debit は `computeOcTotals`（整数円）を使うため整合。ワークシートの統一は別途。
